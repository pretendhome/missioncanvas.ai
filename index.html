How it works

The model is always
the last step.

Every other AI tool sends your query to a model and hopes for the best. Mission Canvas classifies your intent, retrieves evidence-tiered knowledge, and applies governance rules — before any model fires.

SCAN

Entry gate

PII and PHI detected and stripped before classification. Fail-closed.

→
CLASSIFY

Ontology routing

312-node ontology. Keyword → BM25 → LLM fallback. Confidence scored.

→
GOVERN

Architectural enforcement

Sensitive domains block external calls. Not a config. A structural gate.

→
RETRIEVE

Evidence-tiered knowledge

219+ knowledge entries. Freshness-scored. Stale entries filtered before REASON.

→
REASON

Model fires last

Full structured context assembled. Local or cloud — based on governance, not cost.

src/pipeline.py The OS 
# No skipping. No shortcuts.
# The pipeline IS the OS.

def run(query):
  query = scan(query)       # gate: PII
  node  = classify(query)  # ontology
  path  = traverse(node)   # graph walk
  kl    = retrieve(node)   # evidence
  ctx   = context(kl)      # assemble

  if node.blocks_external:
    answer = reason_local(ctx)  # ALWAYS
  else:
    answer = reason(ctx)        # model last

  store(answer)  # path record: always
  return answer
